# Kafka Testing Framework (Java)

Автоматизированный фреймворк для интеграционного и компонентного тестирования асинхронного взаимодействия через Apache Kafka. Проект разработан на стеке Java + Testcontainers + Allure и демонстрирует подходы к валидации асинхронной бизнес-логики, обработке поврежденных сообщений (Poison Pills) и сбору метрик.

## 🚀 Основной функционал проекта
*   **Изолированная инфраструктура**: Автоматический подъем брокера Kafka в Docker-контейнере с использованием актуального модуля `Testcontainers-Kafka` (совместим с Docker Engine 29+).
*   **Гарантия доставки и DLQ**: Тестирование отказоустойчивости сервиса при десериализации битого JSON и проверка синхронного роутинга сообщений в Dead Letter Queue (DLQ).
*   **Асинхронные проверки**: Использование библиотеки `Awaitility` для стабильного ожидания доставки сообщений без использования хрупких `Thread.sleep()`.
*   **Валидация метрик**: Интеграция с `Micrometer (SimpleMeterRegistry)` для контроля счетчиков успешной обработки и ошибок на уровне консьюмера.
*   **Изоляция тестов**: Использование уникальных динамических топиков Kafka для каждого тест-кейса, что гарантирует независимость запусков и исключает взаимное влияние данных.
*   **CI/CD Пайплайн**: Настроенный воркфлоу в `GitHub Actions` для автоматической сборки проекта и прогона тестов в облаке при каждом коммите.

## 🛠 Стек технологий
*   **Язык**: Java 17 (JDK)
*   **Тестовый фреймворк**: JUnit 5 (Jupiter), AssertJ
*   **Работа с Kafka**: Apache Kafka Clients (Producer / Consumer), Testcontainers-Kafka
*   **Асинхронность и метрики**: Awaitility, Micrometer Core
*   **Инструменты**: Lombok, Jackson (JSON ObjectMapper)
*   **Отчетность**: Allure Framework
*   **CI/CD**: GitHub Actions

## 📂 Структура проекта
*   `ru.usetech.handn.model.Order` — Доменная модель заказа с безопасным типом данных для финансовых операций (`BigDecimal`) и паттерном `@Builder`.
*   `ru.usetech.handn.service.OrderConsumerService` — Тестируемый сервис. Реализует чтение из топика, сбор метрик через Micrometer и логику перенаправления поврежденных данных в DLQ.
*   `ru.usetech.handn.KafkaIntegrationTest` — Интеграционные тесты, покрывающие позитивные и негативные сценарии взаимодействия с Kafka.

## 📊 Запуск тестов и генерация отчетов

### Локальный запуск
Для запуска тестов локально необходимо наличие установленного и запущенного Docker Desktop.

1. Сборка проекта и прогон тестов:
   ```bash
   mvn clean test
   ```
2. Генерация и открытие интерактивного Allure-отчета в браузере:
   ```bash
   allure serve allure-results
   ```

### Запуск в CI/CD (GitHub Actions)
Пайплайн настроен в файле `.github/workflows/run-tests.yml`. Он автоматически:
1. Разворачивает окружение на чистой ОС Ubuntu.
2. Инициализирует Docker Daemon.
3. Прогоняет тесты через Maven.
4. Сохраняет результаты тестов (`allure-results`) в виде артефакта сборки.
